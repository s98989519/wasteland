# 廢土迴響 - 網頁 RPG 專案計畫 (Project Plan)

## 核心構思與遊戲架構
本計畫書記錄了專案初期的核心構思與後續確立的遊戲設計方向。

### 1. 核心玩法 (Core Gameplay)
*   **無職業設定 (Classless System)**：玩家初始沒有固定的職業限制。角色的成長與戰鬥流派（Build）完全取決於單局遊戲（Run）內所獲得的裝備、道具與技能選擇。
*   **章節推進 (Chapter-based Progression)**：遊戲以「章節」為單位進行推進（目前第一章為 50 步）。每個章節有最終 BOSS 坐鎮。
*   **抉擇影響 (Consequence of Choices)**：玩家在探索途中做出的事件選擇，未來將會影響到遭遇的 BOSS 類型、難度或機制（例如：幫助了某個NPC，BOSS戰可能會有援助；或是錯誤的選擇會讓BOSS變得更狂暴）。

### 2. 局外養成機制 (Meta-Progression)
*   **大廳 / 主城鎮系統**：遊戲設有「獵人小屋」作為玩家的安全屋與大廳。
*   **跨局成長**：玩家在探索（Expedition）中獲得的經驗、特殊物資（如舊世幣、素材）可以帶回大廳。
*   **能力解鎖**：這些資源可用於解鎖永久的能力點數、起始道具、或特殊加成，讓玩家在後續的輪迴（Run）中體驗變得更好，或是能夠嘗試全新的遊玩流派（Build）。

### 3. 戰鬥風格 (Combat Style)
*   **回合制戰略**：採用行動隊列與 AP/回合制的設計。
*   **動作與意圖**：敵人的下一步動作意圖會提前顯示給玩家，玩家需要根據敵人的意圖來決定本回合要採取「攻擊」、「防禦」、「閃躲」還是「準備技能」。
*   **資源管理**：血量（HP）不會隨便自然回復，且戰鬥中需要考量護甲（Armor）的疊加與消耗。

### 4. 介面與互動設計 (UI/UX Design)
*   **戰鬥介面**：敵人的各種狀態（如：護甲、狂暴層數、縮地閃避等）會並排顯示於敵人名稱的下方，並保留點擊互動功能，點擊後會顯示獨立的說明面板，方便玩家隨時查閱狀態的具體效果與疊加層數。
*   **未知的遭遇**：遭遇敵人時不會提前揭露怪物陣容，只會顯示「前方有威脅」的警告，且不可逃跑，增加廢土探索的壓迫感。
*   **死亡與結算**：玩家在探索或戰鬥中血量歸零時，會觸發「失去意識」的專屬 UI 結算面板。確認後才會返回獵人小屋，並會扣除一半的舊世幣作為死亡懲罰。

### 5. 開發技術棧
*   **前端架構**：純前端網頁遊戲 (HTML / CSS / Vanilla JavaScript)。
*   **模組化設計**：將系統切分為獨立的 Manager / System 類別（如 `GameController.js`, `CombatSystem.js`, `InventorySystem.js`, `EventSystem.js`, `UIManager.js`）。
*   **本地存檔**：（規劃中）未來將使用 `localStorage` 或 `IndexedDB` 儲存玩家的局外養成進度。
